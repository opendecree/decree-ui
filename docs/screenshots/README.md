# Screenshots

Generated by `scripts/screenshots.ts` for use in READMEs and docs.

## Regenerate

1. Start the decree backend with fixtures loaded:

   ```sh
   # from the decree/ repo
   make dev-stack  # or: scripts/dev-stack.sh up && scripts/dev-stack.sh seed
   ```

2. Start the UI dev server:

   ```sh
   # from decree-ui/
   npm run dev
   ```

3. Run the screenshot script:

   ```sh
   npm run screenshots
   ```

PNGs land in this directory — one per page, per color scheme (`light-*` and `dark-*`).

## Contract

Two things the script depends on — don't break them without updating `scripts/screenshots.ts`:

1. **Fixture names** (`billing`, `showcase`, `acme`, `demo`) — resolved to UUIDs via `GET /v1/schemas`
   and `GET /v1/tenants`. Names are stable across seed runs; UUIDs are not.
2. **Page root `data-testid` anchors** — used to wait for the page to render before snapping, so
   screenshots don't capture loading spinners. Anchors in use:
   - `home-page`, `schema-list-page`, `schema-detail-page`, `tenant-list-page`, `tenant-detail-page`

## Environment

| Variable   | Default                 | Purpose                |
| ---------- | ----------------------- | ---------------------- |
| `BASE_URL` | `http://localhost:5174` | UI dev server URL      |
| `API_URL`  | `http://localhost:8080` | decree backend URL     |
| `OUT_DIR`  | `docs/screenshots`      | PNG output directory   |
